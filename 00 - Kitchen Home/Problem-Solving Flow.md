# Problem-Solving Flow

Run this before writing code.

## 1. Ingredient scan

```text
What keywords appear?
What data structures appear?
Is the input sorted?
Is the task contiguous?
Is it graph/tree/grid?
Is it asking for min, max, ways, kth, shortest, or all possibilities?
```

## 2. Constraint taste test

Open [[Constraint Taste Test]] and ask what complexity the input permits.

## 3. Recipe guess

Say this out loud or write it in the problem card:

> This looks like ___ because ___.

## 4. Confusable check

Before coding, ask:

- Could this be sliding window or prefix sum?
- Could this be DFS or BFS?
- Could this be greedy or dynamic programming?
- Could this be binary search on answer rather than direct simulation?

Use [[Confusable Ingredients]] when stuck.

## 5. Cook from template

Open the pattern note from [[Recipe Index]], copy the core template mentally, and adapt only the moving parts.

## 6. Study card

After solving, fill [[Template - Problem Study Card]]. The goal is recognition, not trophy collection.
